# --- ML / DL core ---
# PROD.7: TensorFlow tem 3 distribuições:
#   - `tensorflow` (default): tenta carregar CUDA libs em runtime; ~700MB
#     com bibliotecas GPU. Funciona em CPU mesmo sem GPU, mas warning verboso.
#   - `tensorflow-cpu`: build menor (~250MB), SEM CUDA — recomendado para
#     CPU-only builds (Docker padrão sem GPU). Para usar:
#         pip uninstall -y tensorflow && pip install 'tensorflow-cpu>=2.16,<2.22'
#   - `tensorflow[and-cuda]`: instala CUDA wheels (Linux NVIDIA). Use no
#     docker-compose.gpu.yml override OU em hosts NVIDIA bare-metal.
# Padrão usa `tensorflow` para compatibilidade ampla com CPU + GPU sem reinstalar.
#
# BUG.Install: o upper bound era `<2.20` mas TF 2.16-2.19 NÃO TÊM wheels para
# Python 3.13 — só 3.9-3.12. Resultado: `ERROR: No matching distribution`.
# TF 2.20+ é a primeira versão com suporte Python 3.13. Subimos para <2.22
# para cobrir 3.11-3.13. Keras 3 continua nativo em todas essas versões.

keras>=3.14.1,<4.0
scikit-learn>=1.9.0,<2.0
